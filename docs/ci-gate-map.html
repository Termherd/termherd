The CI gate map

Every push runs only the checks its diff can affect, and they all fan into one required status check, ci-success. That single check is what main branch protection pins; everything else is a signal or a post-merge baseline.

Outside the PR gate

cross-os signal
clippy + tests on macOS & Windows. Skipped on PRs to keep the merge gate fast.
runs: push→main when rust changed · release tags. Not a merge blocker; a red run does not stop a release.
Analyze (Rust) baseline
CodeQL SAST. Traces a full cargo build (the iced-heavy app crate), so a run is minutes long.
runs: push→main + weekly, never on PRs. Not required; reaches the code seconds after merge.
release · package release
cargo-dist archives + installers and the GitHub Release; desktop bundles attached.
runs: tag push matching **N.N.N*. Validates in plan-mode on PRs.

CodeQL query suite: stays on security-and-quality (the full pack). Trimming to security-extended, which drops the maintainability / quality queries, was weighed for speed. But with CodeQL off the PR path its runtime no longer blocks anyone, so the fuller suite is kept for the coverage.

What runs when

Pull request → main

  • →changes + only the gated ubuntu jobs → ci-success
  • –cross-os and CodeQL are skipped
  • –release runs plan-validation only

Merge / push → main

  • →the PR gates re-run on the merged commit
  • +cross-os (mac · win) baseline
  • +CodeQL Analyze (Rust)

Release tag

  • →release + package build & publish
  • +cross-os runs (forced on tags)
  • –CodeQL does not run on tags

Weekly · Mon 07:00 UTC

  • →CodeQL only; catches query-pack drift
  • –flags new findings on code already on main
